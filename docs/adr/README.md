# ADRs (Architecture Decision Records)

Use ADRs to capture important technical decisions for this repository.

## Naming

- `ADR-0001-title.md`
- `ADR-0002-title.md`

## Suggested template

```md
# ADR-000X: Short Decision Title

## Status

Accepted | Superseded | Deprecated

## Context

What problem are we solving and what constraints matter?

## Decision

What choice was made?

## Consequences

What are the tradeoffs, risks, and follow-up work?
```

## Guidance

- Keep ADRs short and concrete.
- Link related PRs/issues.
- Create a new ADR instead of rewriting history.
